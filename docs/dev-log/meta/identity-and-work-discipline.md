# 身份边界与工作纪律

> 个人备忘。与 [ai-collaboration-harness-loop.md](./ai-collaboration-harness-loop.md)、[开发地图](../../guide/intro/roadmap.md) 并列阅读。

---

## 为什么两条线一起走

- **旧身份仍在**：本行（专利 → 桌面摆件合作 → 论文）是 **一条河**，不是与 Pomotention 抢时间的杂项。
- **下午 Pomotention**：新产品、新身份、可能的被动收入；**不可能一下子做好**。
- **只做 Pomotention 会焦虑**：收入、认可、自我认同仍在本行；转型若像「全盘否定过去」，心里会空。
- **目标**：约 18 个月后，本行还在，同时 **有产品、收过钱、开发者身份实存**——不是一夜爆款，是双线各走半步、走很多步。

**只记一句：**

> 不是「选旧还是选新」，是「旧身份供养过渡，新身份每天下午实存」。

---

## 两条线分别是什么

| | **本职工作（一条河）** | **Pomotention** |
|---|------------------------|-----------------|
| **内容** | 专利 → 摆件原型/合作 → 论文（测试依赖产品，**IndexedDB 之后**再开论文段） | ledger → 收费 → schema → IndexedDB… |
| **时段** | **上午 2h**（+ 原工作其余部分） | **下午 1:30–5:00，3h** |
| **节奏** | 专利约 **1 个月收束**；之后上午给摆件，**不会空出来** | 长期；允许慢 |
| **关系** | 专利、摆件、论文是 **同一身份上的阶段**，不是三条并列副业 | 独立一线 |

**日程骨架：**

```text
6:30–9:30   漂浮：整理、看书、洗漱（不强行产出）
上午 2h     本职工作
下午 3h     Pomotention（1:30 起）
晚上        画画 / 电视 / 游戏（恢复，必要）
```

## 纪律分层（从外到内，顺序不能乱）

### 第一层 · 锚点（最重要）

| 规则 | 含义 |
|------|------|
| **上午只干本行** | 专利 / 摆件 / 论文（按阶段） |
| **下午只干 Pomotention** | 不拿本行材料占下午 |
| **上午没做完 → 不准下午补** | 下午也不准侵占上午 |
| **下午没做完 → 不准晚上狂补** | 记明天下午第一条 |

这是在守 **身份边界**，不是守「今日任务全清」。

**变动大十字**在这里的用法：不是「两周只干一个项目再换」，而是 **块与块之间要稳、块内可以有变化**。上下午切分 = 稳；块内每日小目标 = 变。

**「一次满足一个方面的需要」** 落在三个层面 **同时** 满足，而不是周周换人生：

| 需要 | 怎么满足 |
|------|----------|
| 安全 / 旧身份 | 上午本行 + 原工作 |
| 创造 / 新身份 | 下午 Pomotention |
| 恢复 / 不耗竭 | 漂浮时段 + 晚上娱乐 |

---

### 第二层 · 半天内：一个小目标 + 一个小产出

在 **当前半天** 里再定：

- **今天只做 X**（一个）
- **怎样算停**（验收 1～2 条）

例：

```text
【上午】今天只做：专利权利要求某一节
      停的标准：该节可发给合作者过目

【下午】今天只做：ledger 复制按钮
      停的标准：样例文本能复制 + pnpm build:fast 过
```

没做完 → 写入 **该半天的明日第一条**，不跨半天补。

---

### 第三层 · 一周（可选，服务第二层）

- **这周下午主题** 只打一个（例：ledger 周 → 收费周 → schema 周）。
- **不用**「两周只做产品、再两周只做专利」——整块抽离会 **忘得厉害**。
- 本行 **每天上午 2h** 保持手指记忆；专利月收束后 **上午交给摆件**，不是释放给论文或产品。

---

## 「次」定在哪（避免搞不清）

| 粒度 | 定什么 | 不要用来 |
|------|--------|----------|
| **半天** | 身份边界（本行 / Pomotention） | 半天里再开第三条线 |
| **一天** | 该半天内的一个小产出 | 跨半天补作业 |
| **一周** | 下午 Pomotention 的一个主题 | 切换「整个人只做一件事」 |
| **一月** | 本行内部阶段（专利月 → 摆件月） | 用月来取消上下午切分 |

---

## 下午 1:30 · 十分钟重启（专治「一开就挺不住」）

不靠意志力，靠 **外部记忆 + 切刀**：

1. 打开 [`current.md`](../../current.md) 或 [roadmap 队列](../../guide/intro/roadmap.md)  
2. 写 **今天只做 X**  
3. 写 **停的标准**（1～2 条）  
4. 开干  

模板：

```text
日期：
下午主题（本周）：________
今天只做：
停的标准：
昨天停在哪：（一行）
```

与 AI 协作时，可把「停的标准」直接当 [Harness / 小 Loop](./ai-collaboration-harness-loop.md) 的验收条。

## 本行未竟事项怎么处理

| 情况 | 做法 |
|------|------|
| 上午专利/摆件没做完 | 记 **明日上午第一条**；下午 **照常** Pomotention |
| 下午 Pomotention 没做完 | 记 **明日下午第一条**；**不** 用晚上或上午补 |
| 论文还没开始 | 正常；门槛 = **IndexedDB 之后 + 产品可测**，不写 guilt list |
| 摆件卖不卖未定 | 先 **原型机**；和 Timer/收费一样，闭环后再叙事 |

**论文**在本行那条河里，排在 **IndexedDB 之后**；不占下午主题，除非 deliberately 把某周上午让给写作。

---

## 只记三句

1. **上下午 = 两种身份；混了比没做完更伤。**  
2. **没干也不准跨半天补——这是锚点，比 todo 重要。**  
3. **再在半天里要一个小产出；一周只定一个下午主题。**
