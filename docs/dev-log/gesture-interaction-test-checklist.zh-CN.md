# 手势与点击回归测试清单

用途：改.planner / 活动表 / 时间表拖拽 / 触摸逻辑后，按平台逐项勾选。  
相关实现说明见 [gesture-interaction-inventory.zh-CN.md](./gesture-interaction-inventory.zh-CN.md)（若与代码不一致，以代码为准）。

---

## 环境与账号

- [ ] **桌面**：Chrome / Edge（Windows）
- [ ] **iPhone**：Safari（实机）
- [ ] **Android**：Chrome（实机）
- [ ] 横竖屏各扫一眼（至少 **Timetable**、**活动多列**）

---

## 年视图（Year Planner）

- [ ] **桌面**：单击当月某日 → 选中该日（应用日期 / 任务侧联动符合预期）
- [ ] **桌面**：双击当月某日 → 进入日视图（或等价路由）
- [ ] **iPhone / Android**：单击某日 → 仅选日（无长按菜单误触可接受）
- [ ] **iPhone / Android**：快速双触某日 → 进入日视图
- [ ] **iPhone / Android**：单击后约 300ms 内无第二次轻触 → 不应误判为双触
- [ ] 邻月灰格 / 空位：不可点或行为与改造前一致

### 年视图 — 滚动与误触（重点）

- [ ] **iPhone / Android**：在年视图上 **快速上下滑动**（手指落在某日格子或周号附近起滑）→ 页面 **顺畅滚动**，松手后 **不应** 切换选中日期、**不应** 误进日视图
- [ ] **iPhone / Android**：滑动过程中略经某日格 → 仍视为滚动，**不** 触发选日
- [ ] **iPhone / Android**：在格子上 **轻触无位移**（或位移极小）松手 → 仍应正常 **单触选日 / 双触进日视图**
- [ ] **桌面**：滚轮 / 拖拽滚动条浏览年视图 → **不** 误改选中（除非点了日期格）

---

## 月视图（Month Planner）

- [ ] 点击 **日期 badge（数字+色块）** → **选日**（与年视图选日语义一致）
- [ ] **桌面**：双击 badge → **进日视图**
- [ ] **iPhone / Android**：双触 badge → **进日视图**
- [ ] 点击 **列表 item**（带标签/时间那条）→ **选中该行 / item**；**不**应冒泡成「只选日、不选 item」
- [ ] 点击卡片空白（非 badge、非 item）→ 选日（若产品仍保留 `day-card` 整卡点击）

---

## 日待办（Day Todo）— 意图列

- [ ] **桌面 / 移动**：单击意图格 → **直接进入标题编辑**（含选中行、光标进输入框）
- [ ] 单击行内其他列（时间、勾选等）→ 选中行、**不**误开标题编辑（除非该列本来就是编辑列）
- [ ] 意图格 **无** 独立「移动双触才编辑」：单触即应一致

---

## 日日程（Day Schedule）— 意图 / 地点

- [ ] 单击 **意图** 格 → 编辑标题
- [ ] 单击 **地点** 格 → 编辑地点
- [ ] 与 Day Todo 相同：无「先延迟选行再双击」路径

---

## 活动清单（Activity Section）

- [ ] **移动**：单击标题区（非 `.icon-tag`）→ **进入标题编辑**
- [ ] 点击 **标签图标** → 仍正常（不被标题 `preventDefault` 吃掉）
- [ ] **拖拽柄列 / 最后一列**：长按或拖动 → **仅调整顺序**，不误进编辑
- [ ] 父节点有子项时：**拖拽 vs 点标题** 仍分得开（阈值/柄与标题区域）

---

## 时间表 Timetable — Todo 段拖拽

- [ ] **桌面**：在 todo 段上 **轻点** → **不**应进入拖拽态（或可与 Popover 等点击共存，以当前设计为准）
- [ ] **桌面**：按下并移动超过约 **十几像素** → 进入拖拽，可放到空番茄格
- [ ] **Android**：轻点段 → **尽量能点开 Popover / 工具提示**，而非一上来被当成拖
- [ ] **Android / iPhone**：明确 **拖移** → 仍能放下、数据 `globalIndex` 更新
- [ ] 拖的过程中 **页面不横向乱滚**；拖完 **滚动恢复**

---

## Naive UI / 合成点击

- [ ] `trigger="click"` 的 **Popover / Dropdown**（排期条、Timetable 移动触发器等）在 **三端** 仍能打开/关闭
- [ ] 无整页莫名 **第二次才生效** 的点击（若出现，记具体页面与步骤）

---

## 回归冒烟（与手势弱相关但易漏）

- [ ] 从各 Planner **切换 Tab** 后日期、选中任务仍合理
- [ ] **登录 / 同步 / 刷新** 后同一时间格、活动顺序不乱
