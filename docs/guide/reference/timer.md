## 单次计时模式 (Pomodoro Mode)

适用于单次专注或单次休息。  
<img src="/pomodoro-mode.png" alt="Pomodoro Mode" width="200">

- **开始工作**：点击 `Wind up` 启动工作周期。
- **取消工作**：点击 `Squash` 终止当前工作计时。
- **开始休息**：点击 `Break` 启动休息计时。
- **结束休息**：点击 `Stop` 可提前结束休息。
- **调整休息时长**：支持 `02` / `05` / `10` / `15` / `30` / `60` 分钟。

### 结束后继续正计时

在 [计时设置](#计时设置) 中开启 **「结束后继续正计时」** 后：

- 倒计时到 `00:00` 时**不会立刻结束**，时钟改为从 `00:01` 起**正计时**，显示超出计划时长的部分。
- 计划时长内：`Squash` 仍表示提前放弃当前工作（记作废 🥫）。
- 进入正计时后：工作按钮变为 `Stop`，按 **实际总时长**（含超出部分）计入统计。
- 休息中始终为 `Stop`；正计时阶段结束休息，同样按实际总时长记录。

未开启时，到点自动结束并复位，与原先行为一致。

### 声音提示节奏（25 分钟示例）

- 工作开始提示；
- `Review` 阶段（2 分钟）提示；
- 两段 `Work` 阶段（各约 10.5 分钟）提示；
- `Review` 阶段（1 分钟）提示；
- `Track` 阶段（1 分钟）无声音提示；
- 周期结束提示。

### 休息提示

- 开始；
- 1/5；
- 2/5；
- 3/5；
- 结束。

## 连续计时模式 (Pizza Mode)

适用于多个工作/休息周期连续执行。  
<img src="/pizza-mode.png" alt="Pizza Mode" width="200">

- **模式切换**：点击 🍅🍕 切换单次/连续模式；运行中不可切换。
- **自定义序列**：在输入框中填写序列指令：
  - `🍅+05`：一个工作周期后自动休息 5 分钟。
  - `05+05`：连续两个 5 分钟休息段。
- **修改默认工作时长**：通过 `🍅=25min` 调整，支持 `15min` 到 `59min`。
- **开始序列**：点击 ▶（Play）启动；与单次模式的 `Wind up` 一样会记入一次「开始工作」操作。
- **提前停止**：点击 ⏹（Stop）终止当前段并结束整轮序列；工作中记为 Squash，休息中记为 Stop。
- **白噪音**：点击 🔈 开关背景音，点击红点切换滴答/雨声。

### 提前停止与确认

连续模式下，**每一段自然跑完**时会立刻写入一条统计；**尚未开始的后续步骤不会产生记录**。

例如序列 `🍅+05+🍅+05+🍅`：若前两个 🍅 和两个 05 已跑完，第三个 🍅 中途按 Stop，则统计里已有 4 条完成记录，当前段再按 Stop 规则处理；第五个及之后的步骤从未开始，**不会有任何记录**。

| 情况 | 是否弹窗 | 结果 |
|------|----------|------|
| 当前段从未完整跑完就 Stop（如第一个 🍅 只跑了 15 分钟就停） | 否 | 与单次 Squash 相同，记 **1 条作废 🥫** |
| 本轮已有至少 1 段 **完整完成** 的 work 或 break | 是 | 见下表 |

弹窗问：**「已完成的番茄/休息是否计入统计？」**

| 你的选择 | 统计结果 |
|----------|----------|
| **计入** | 保留已完整完成的各段；当前被打断段：工作按**该段开始时的计划时长**分档，休息按实际时长 |
| **不计入** | 撤销本轮已完整完成的记录，整轮视为一次放弃，仅保留 **1 条作废 🥫** |
| 关闭对话框 / Esc / 点遮罩（未明确点「不计入」） | **默认按「计入」处理并停止**（避免误删已完成的记录） |

**示例** `🍅+05+🍅`：第一个 🍅 与 05 已跑完，第二个 🍅 中途 Stop：

- 选 **计入** → 🍅、☕、🍅（第三个按当时 🍅 设定分钟数分档）
- 选 **不计入** → 仅 🥫

## 统计与记录

Timer 独立壳中的 **统计** 页展示每周 emoji 与时长汇总；**规则** 对话框可调整分档阈值与 emoji。

### 何时写入一条记录

- **开始**一段 work 或 break 时，在内存中打开一段会话。
- **自然跑完**或 **Stop / Squash** 时，写入一条持久记录。
- 序列中**未开始的步骤**不会预先占位。

### 结束方式与含义

| 场景 | 统计表现 |
|------|----------|
| 工作自然完成 | 按**实际时长**分档（🍒 / 🍅 / 🍉 等，依规则） |
| 工作 Squash（计划内提前放弃） | 作废 🥫，不计工作时长 |
| 工作 Stop（含「结束后继续正计时」超时后停止） | 按**实际总时长**分档（如计划 25min 跑到 45min，按 45min 计） |
| 休息自然完成或 Stop | 按实际时长计 short / long break |
| Pizza 提前停、选「不计入」 | 撤销本轮已完成段 + **1 条 🥫** |

**分档 emoji 回落**：统计始终用真实分钟数；若自然档位在规则里**未勾选**计入，则向下降落——例如 Long Work 未勾选时，45min 的工作仍显示并计入 **Work 🍅**，而不是 Long Work emoji。

**规则对话框**：Squash、Work、Break（short）三档**不可取消勾选**；Short Work、Long Work、Long Break 可选是否计入图表。

悬停统计行中的 emoji 时，若该段执行时有自定义状态文案，会显示为提示；无文案则不显示。

## 计时设置

在 Timer 壳 **设置** 页可调整：

| 项 | 说明 |
|----|------|
| 工作 / 休息时长 | 单次模式默认计划时长 |
| 结束后继续正计时 | 见 [单次模式](#结束后继续正计时) |
| 分段提示音 | 中间阶段切换与休息节点提示；开始/结束音不受影响 |
| 白噪音 | 工作时的背景音开关与音轨 |

统计分档（工作/休息分钟阈值、emoji、是否计入图表）在统计页 **规则** 中单独配置，与工作时长设置相互独立。

## 迷你计时器模式 (On top Mode)

为低干扰专注场景设计：

1. 点击菜单栏 <img src="/icons/Pin24Regular.svg" width="20" style="display:inline-block;vertical-align:middle;margin:0;background:rgb(224, 224, 224);border-radius: 6px"> 进入置顶迷你模式。
2. 点击计时器界面 <img src="/icons/ArrowExpand24Regular.svg" width="20" style="display:inline-block;vertical-align:middle;margin:0;"> 退出迷你模式。
3. 支持多实例同时运行，并可分别设置是否迷你显示。
4. macOS 可结合 Sidecar 将迷你计时器放到 iPad 等扩展屏。

## 限制与说明

- 计时运行时不能切换 `单次/连续` 模式。
- 某些平台可能偶发窗口尺寸计算异常，重新进入迷你模式可触发自动修正。
