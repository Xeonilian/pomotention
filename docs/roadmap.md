# 开发地图

> 目标：用统一尺度描述计划，避免“版本/主题/任务/时间”混写。

## 使用方式（统一尺度）

- **主题（Theme）**：长期方向（为什么做）
- **史诗（Epic）**：一组能独立验收的能力（做什么）
- **可交付项（Deliverable）**：一条可完成/可验证的具体事项（怎么验收）
- **状态**：
  - ✅ 已完成
  - 🚧 进行中/搁置
  - 📝 待规划
