Pomotention Timer · 14 天锚定单 ​

落地前问卷结论（2026-05-25）。执行 Timer 独立 Win 包冲刺时以此为准；与 RELEASE-dev 主站发布流程并行、互不替代。

一句话 ​

在 14 天内，为 「只要电脑专注计时」的用户（从现有 60 人 / 社群等自然来即可）交付 Windows Timer 安装包；验证 累计 ≥5 次完成工作段（A-松，可不点名）；为此 暂不做收费、不做手机 Timer 专用 PWA、不做登录同步；完整版 Pomotention 保持免费，Timer 通过中性脚注引流完整版。

14 天后怎样算赢 ​

项约定
主指标（A-松）对外发出 Win Timer 安装包；本机统计 累计 ≥5 次「工作段完成」
不要求事先点名 5 人、逐一访谈
建议自己先用顺；完成数中应有一部分来自「丢出去」之后，而非仅自测
对外主交付Windows 安装包（必达）
可选 bonus无签名 Mac .dmg（有人能装即可；不为 Mac 单独耗一周）
这两周不收款（Gumroad 等放下一轮）

产品定义（独立完整小产品，非阉割版） ​

  • 包含：单次番茄 + 休息、Pizza 番茄列车、白噪音、置顶/迷你窗、顶上可编辑状态文案（pomodoroStateMessage）、Timer 相关设置（时长、声音、紧凑等）。
  • 不包含：planner / 任务 / 登录 / 云同步 / 与完整版 planner 数据互导；顶栏跑马灯「今日目标」。
  • 统计（档 1）：本机 今日完成工作段数 展示；不做 导出、点击流、会话列表（档 2 以后再说）。
  • 置顶窗：不进设置；先退出置顶再改；单个番茄模式也要能开关白噪音（置顶时仍可操作）。
  • UI 壳：保留现网 顶栏 = 置顶 Pin + 设置；去掉其它视图切换、数据库、登录等；设置页只保留 Timer 相关项。

对外一句话（桌面包）

电脑工作时窗口置顶一直看见；番茄列车帮你连续跑一整组专注；自带白噪音，番茄长短也能自己设。

与完整版 Pomotention

项约定
关系Timer 可引流完整版（c）；完整版 继续免费（完整版若未来收费 → 倾向 订阅；Timer 若未来收费 → 倾向 一次性买断，非订阅）
页脚中性脚注（a）：「需要日程与任务？打开完整版 Pomotention →」
以后Timer 数据迁完整版的「大升级」单独立项，不进本 14 天

用户、发布与支持 ​

项约定
用户池已注册 ~60 人、番茄学习群、同事、小红书等；丢出去即可，不凑 5 个实名制
支持一页安装/置顶/白噪音说明；一个主反馈渠道；文字回复；每天约 30 分钟 上限
热情过载可用模板：「感谢！这轮我在收集中，会先修共性问题；你的情况我记下了，X 天内回复。」
最怕白费两周时间在主项目 新功能 / 异想天开，没朝 Timer Win 包推进

时间与主项目 ​

项约定
投入平均每天 ≥3 小时
主项目基本停更（c）；不修新功能；会修 现网 blocker（如换天 bug）
第 7 天检查点若尚不能给别人装 → 砍 scope（a），先「能装 + 能跑完一轮工作段」

每天自检

今天有没有让「多一个人能装上并完成一轮工作段」更近？

技术路线 ​

  • 同 repo，不新 repo：Timer 瘦入口 + 第二套 Tauri 构建（独立 identifier / 产品名等）；不必为打包单独维护长期分支。
  • Win 必达；Mac 无签名 dmg 可作 bonus；不做 Timer 专用手机 PWA 子域、应用商店、Mac 签名/公证。

建议 14 天节奏 ​

阶段天数焦点
11～3Timer 瘦壳：仅钟 + 瘦顶栏；tauri build 出 Win 包；自己跑通 1 轮
24～6置顶可靠；单次模式白噪音；Timer 设置瘦身；档 1 今日计数
37检查点：能否给别人装？否 → 按 砍 scope 顺序
48～10一页说明；丢到主渠道；累计 ≥5 工作段
511～14修 blocker；脚注链完整版；可选收集 3 条非己反馈原话

第 7 天砍 scope 顺序 ​

从下到上砍；砍前一层仍算检查点未通过。

  1. 档 1「今日 🍅 计数」UI
  2. 单次番茄模式白噪音按钮（若列车模式仍可用白噪音则暂可砍此项）
  3. Pizza 序列（保留单次番茄 + 休息）
  4. 设置页精简（暂用默认时长/声音，仅保留置顶与核心按钮）
  5. Mac dmg（只发 Win）

不可砍：Win 能安装、能完成 至少 1 次工作段（Wind up → 正常结束或 Squash）、置顶可用。

两周绝对不做 ​

  1. 新 repo
  2. iPhone / Android Timer 专用 PWA 子域（提前做完再说不算本盒）
  3. 应用商店上架
  4. 收费 / Gumroad / 标价落地
  5. 登录、云同步、与完整版 planner 数据互导
  6. 会话导出、点击流日志（统计档 2）
  7. 主项目 新功能
  8. Mac 签名 / 公证（无签名 dmg 仅作 bonus）
  9. 系统级专注锁（引导式访问等仅可写文档提示）

价值观与商业化（长期红线） ​

不做：广告；卖数据/行为追踪换钱；复杂 DRM；Timer 订阅制；阉割 Timer 逼用户上完整版（灰掉 tab 类）。

在乎：可持续收入 + 发布/反馈闭环；低客服是底线（非炫耀目标）。

独立开发者日课（本 14 天）

我今天只承诺：30 分钟回 Timer 安装/闪退；功能想法记 backlog；14 天不收钱。

第 15 天以后（刻意不计入赢） ​

  • Gumroad（或类似）+ Timer 一次性买断 + 自动发 Win 包；熟悉流程即可，不强求首单。
  • Timer → 完整版数据迁移。
  • 手机 Timer PWA / 替代实体钟。
  • 完整版订阅（若做）。

用 Pomotention 管本 sprint（与软件内 backlog） ​

  • 长期 #list 活动 + Task 内 - [ ] 可继续用；冲刺另开 Timer 14d 无日期活动，避免与百年待办混页。
  • Task 清单用 [now] / [later]；计划表 今天 只放 [now] 里 1～3 条。
  • 清单过长：优先 Task 内「收件箱 / 已完成」分区；主题或阶段变了再 新开活动，旧条 archive。
  • 操作说明见 长期事项与待办库存。
